# Surikata

Surikata is a minimalist, mobile-first, multi-user RSS reader. Users register, subscribe to preset or custom feeds, and read all their articles in one unified, filterable feed — with read/unread tracking, Read Later, and Favorites.

Built with [Hono](https://hono.dev) on Cloudflare Workers, using Cloudflare D1 (SQLite) for storage and a Cron Trigger to refresh feeds in the background.

## Features

- Unified login/register page, PBKDF2-hashed passwords, signed JWT session cookie.
- Home feed with source filter chips, "Hide Read" toggle, "Read Later" and "Favorites" tabs.
- Article cards with source, title, perex (truncated ~200 chars), optional image, bookmark/star actions.
- Clicking an article opens it in a new tab and marks it read in the background.
- "Mark all as read" for the current filtered view.
- Admin page to toggle preset feeds (Deník Alarm, Druhá směna, Page Not Found) and add custom feeds by pasting a website URL (auto-discovers `<link rel="alternate">` feed tags) or a direct RSS/Atom URL.
- Background feed refresh via a Cloudflare Cron Trigger (every 15 min) — the homepage only reads from D1, no live fetch on page load.

## Project Structure

```
src/
  index.ts            Hono app entry + scheduled() export
  scheduled.ts         Cron handler: fetches all subscribed feeds into D1
  auth.ts              PBKDF2 password hashing + JWT session signing (Web Crypto only)
  types.ts             Shared Env/type definitions, preset feed list
  middleware/session.ts  Session-loading + auth-gate middleware
  rss/parser.ts        RSS 2.0 / Atom XML parser
  rss/discover.ts      <link rel="alternate"> feed autodiscovery
  routes/auth.ts       /login, /register, /logout
  routes/feeds.ts       /, /partials/feed, /articles/*
  routes/admin.ts      /admin, /admin/feeds/*
  views/layout.ts      Shared HTML shell (top nav desktop / bottom nav mobile)
  views/article-card.ts  Article card HTML renderer
  styles/input.css     Tailwind entry point (compiled to public/styles.css)
public/
  app.js               Vanilla JS: filter chips, toggles, bookmark/favorite, admin interactions
  styles.css           Generated by `npm run build:css` — do not edit directly
schema.sql             Reference schema (mirrors migrations/0001_init.sql)
migrations/0001_init.sql  D1 migration applied via wrangler
```

## Local Development

Requires Node.js 18+.

```bash
npm install

# Build the Tailwind CSS once (or use watch:css while developing)
npm run build:css

# Apply the schema to a local D1 database (SQLite file under .wrangler/state)
npm run db:migrate:local

# Start the local dev server (wrangler dev --local)
npm run dev
```

The app will be available at the URL Wrangler prints (typically `http://localhost:8787`). Register a user, then visit **Manage Sources** to subscribe to preset feeds or add your own.

Note: locally, articles only populate after the Cron Trigger runs. Wrangler's local dev does not auto-fire cron on a schedule — trigger it manually while developing:

```bash
curl "http://localhost:8787/__scheduled?cron=*+*+*+*+*"
```

(Wrangler exposes this test endpoint for scheduled handlers in local dev.)

While iterating on styles, run `npm run watch:css` in a second terminal to rebuild `public/styles.css` on change.

## Cloudflare Deployment

1. **Create the D1 database:**
   ```bash
   npx wrangler d1 create surikata-db
   ```
   Copy the returned `database_id` into `wrangler.toml` under `[[d1_databases]]`.

2. **Apply the migration to the remote database:**
   ```bash
   npm run db:migrate:remote
   ```

3. **Set the JWT signing secret** (replace the insecure dev default in `wrangler.toml`):
   ```bash
   npx wrangler secret put JWT_SECRET
   ```

4. **Build the CSS bundle:**
   ```bash
   npm run build:css
   ```

5. **Deploy:**
   ```bash
   npm run deploy
   ```

The Cron Trigger defined in `wrangler.toml` (`*/15 * * * *`) will automatically start refreshing subscribed feeds on the deployed Worker — no extra setup needed.

## Database Schema

See `schema.sql` / `migrations/0001_init.sql`:

- `users` — id, username, password_hash, password_salt
- `user_feeds` — user_id, feed_url, feed_name (PK: user_id + feed_url)
- `articles` — shared cache of parsed feed items, populated by the cron job
- `read_articles` — user_id, article_link (PK)
- `read_later` — user_id, article_link, title, source, perex, image_url
- `favorites` — user_id, article_link, title, source, perex, image_url
